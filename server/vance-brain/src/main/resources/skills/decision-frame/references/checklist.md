# Decision-Framing Checklist

A short list to run through before declaring a decision "framed". Don't recite this verbatim to the user — use it as your own guardrail.

## Scope

- [ ] Is the decision **stated as a question** with a definite verb? ("Should I X?" not "I'm thinking about X.")
- [ ] Are the **options enumerated** explicitly? Even "do nothing" counts as an option.
- [ ] Have you checked for a hidden third option the user hasn't mentioned?

## Criteria

- [ ] Does the user's **stated criteria** match what they actually care about? Watch for borrowed metrics ("everyone says…", "best practice is…").
- [ ] Is there a **dominant criterion** that overrides the others, or is this genuinely multi-axis?
- [ ] Are any criteria **incommensurable** (money vs. relationships, speed vs. quality)? Flag explicitly — those don't add up cleanly.

## Information

- [ ] Does the user have **enough information** to decide, or are they deciding under fog?
- [ ] If under fog: is there a **cheap experiment** that would resolve the biggest unknown?
- [ ] Are there **stakeholders** whose input would change the answer? Have they been consulted?

## Reversibility

- [ ] Two-way door (cheap to reverse) → favor speed and learning.
- [ ] One-way door (expensive/impossible to reverse) → favor caution and sanity-checks.
- [ ] If unsure: how much would it cost to undo? In time, money, or relationships?

## Bias checks

- [ ] **Sunk-cost** — is the user keeping an option alive because of past investment?
- [ ] **Status quo** — is "stay the same" being given a free pass without scoring?
- [ ] **Recency** — is one option benefiting from a recent positive (or suffering from a recent negative) data point?
- [ ] **Authority** — is the user deferring to someone else's preference disguised as their own?

## Output

- [ ] Did you separate **structure** (the framing) from **answer** (the recommendation)?
- [ ] Is the **next step** clear and actionable, even if the decision itself is "not yet"?
- [ ] Did you **acknowledge uncertainty** instead of false-precise scoring?
